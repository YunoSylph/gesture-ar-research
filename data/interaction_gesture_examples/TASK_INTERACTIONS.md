# Task Interaction Notes

Эта памятка связывает дубликаты видео-жестов из соседних подпапок с действиями в интерфейсе Gesture AR.

## Общие жесты

| Папка | Жест | Действие |
|---|---|---|
| `01_point_2f_pointer_hover` | `point_2f` | Навести красный AR-курсор или reticle на объект, строку, карточку, предмет или точку. |
| `02_click_2f_select_confirm` | `click_2f` | Подтвердить выбранный объект: выбрать, открыть, взять, сбросить, закрепить. |
| `03_swipe_left_previous` | `swipe_left` | Предыдущий элемент, скролл вверх, сдвиг/поворот влево, возврат курсора к исходной зоне. |
| `04_swipe_right_next` | `swipe_right` | Следующий элемент, скролл вниз, перенос предмета вправо, сдвиг/поворот вправо. |
| `05_zoom_in_scale_up` | `zoom_in` | Увеличить объект, карточку или измеренный участок. |
| `06_zoom_out_scale_down` | `zoom_out` | Уменьшить объект или выполнить точную подгонку. |

## Типовые задачи

| Задача | Последовательность | Визуальный эффект |
|---|---|---|
| `AR Scroll List` | `RIGHT -> RIGHT -> LEFT -> CLICK` | Плавающий список смещается вниз/вверх, активная строка подсвечивается и открывается по клику. |
| `Spatial Browser` | `POINT -> RIGHT -> RIGHT -> CLICK -> ZOOM+` | Карточки сдвигаются как AR-карусель, центральная карточка открывается и увеличивается. |
| `Virtual Sorting` | `POINT -> CLICK -> RIGHT -> CLICK -> LEFT` | Предмет берётся из левого контейнера, переносится вправо и сбрасывается в целевой контейнер. |
| `Object Control` | `POINT -> CLICK -> ZOOM+ -> ZOOM-` | Один AR-объект выбирается и меняет масштаб. |
| `Target Selection` | `POINT -> CLICK -> POINT -> CLICK` | Цели подсвечиваются курсором и подтверждаются. |
| `Object Placement` | `POINT -> CLICK -> ZOOM+ -> RIGHT` | Объект якорится на поверхности, масштабируется и ориентируется. |
| `Distance Measure` | `POINT -> CLICK -> POINT -> CLICK -> ZOOM+` | Две точки измерения фиксируются, затем участок приближается. |
| `Precision Docking` | `POINT -> LEFT -> RIGHT -> ZOOM- -> CLICK` | Объект совмещается с reticle и фиксируется. |

## Как использовать при записи локальных видео

Записывайте ровно эти жесты и не добавляйте новые классы в локальный набор. Для будущего phone AR режима смысл жеста остаётся экранным: `LEFT` и `RIGHT` определяют направление действия на экране/в AR-сцене, а не анатомическое направление ладони.

Сторону ладони из webcam-reference повторять не нужно. Запись с задней камеры телефона должна быть естественной для AR: если камера видит тыльную сторону руки, это всё ещё тот же жест. Главное, чтобы пальцы были различимы и действие соответствовало label.
